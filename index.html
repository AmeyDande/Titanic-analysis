FEATURE ENGINEERING DECISIONS
WHY fare_per_person?
→Raw Fare reflects total ticket price for an entire family sharing a ticket
→A family of 5 paying £200 each paid £40/person — not £200/person
→fare_per_person = total_fare ÷ family_size fixes this distortion
WHY Ridge for Age imputation?
→Simple median fill treats all passengers as equal — ignores Pclass, Sex, family context
→Ridge captures soft patterns: 1st class passengers skew older
→Low R² (0.19) is expected — Age is largely independent of available features

FASTAPI DEPLOYMENT — APP.PY
# app.py — FastAPI prediction endpoint on Render import joblib, numpy as np from fastapi import FastAPI from fastapi.middleware.cors import CORSMiddleware from pydantic import BaseModel app = FastAPI() app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"]) model = joblib.load('titanic_model.pkl') class Passenger(BaseModel): Pclass: int; Sex: int; Age: float; SibSp: int; Parch: int total_fare: float; lone_traveler: int; family_size: int fare_per_person: float; Embarked_S: int; Embarked_C: int; Embarked_Q: int @app.post('/predict') def predict(p: Passenger): data = np.array([[p.Pclass, p.Sex, p.Age, p.SibSp, p.Parch, p.total_fare, p.lone_traveler, p.family_size, p.fare_per_person, p.Embarked_S, p.Embarked_C, p.Embarked_Q]]) return {'survived': int(model.predict(data)[0])}
